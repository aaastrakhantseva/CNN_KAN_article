# STL10 CNN + KAN: Сравнение MLP и Kolmogorov–Arnold Network

Исследовательский проект, посвящённый сравнению традиционного полносвязного классификатора (**MLP**) и современной архитектуры **Kolmogorov–Arnold Network (KAN)** для задачи классификации изображений на датасете STL-10.

## О проекте

**Цель проекта** — исследовать эффективность замены традиционного MLP-head на KAN-head при использовании одного и того же CNN-backbone.

### Реализованный функционал

- Извлечение признаков с помощью CNN-backbone
- Классификация изображений двумя подходами:
  - **Baseline**: CNN + MLP Head
  - **Основной**: CNN + KAN Head
- Сравнительный анализ моделей по метрикам качества
- Сохранение моделей, признаков и чекпоинтов
- Визуализация результатов обучения и архитектур моделей

## Архитектура моделей

Проект реализует две архитектуры:

- **CNN Backbone** — извлечение признаков (feature extractor)
- **MLP Head** — традиционный классификатор (baseline)
- **KAN Head** — Kolmogorov–Arnold Network (исследуемый подход)

**Визуализация архитектур:**
- Архитектура CNN Backbone и MLP Head описана с помощью `torchinfo`
- Архитектура KAN визуализирована с помощью **Netron** 

Скриншоты и выводы `torchinfo` находятся в папке `reports/architecture/`.

## Результаты эксперимента

| Метрика                  | CNN + MLP   | CNN + KAN   | Изменение    |
|--------------------------|-------------|-------------|--------------|
| Accuracy                 | 0.6275      | **0.6601**  | +3.26%       |
| Precision                | 0.6518      | 0.6568      | +0.50%       |
| Recall                   | 0.6275      | **0.6601**  | +3.26%       |
| F1-score                 | 0.6272      | **0.6574**  | +3.02%       |
| Финальный Train Loss     | 0.8799      | **0.206**   | Существенное улучшение |

**Вывод:** Замена MLP-head на KAN-head позволила улучшить качество классификации без изменения backbone-сети.
